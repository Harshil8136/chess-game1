3) Data model — pre-generated monthly files

You said you want pre-generated month files (“Gemini will create whole code for a specific separate month”): that works perfectly. Each file is a plain JS file that defines a global schedule object — no JSON, no fetch required.

File naming convention
/src/data/calabrio-YYYY-MM.js or /src/data/calabrio-oct-2025.js

Data structure (example schema)

/* calabrio-2025-10.js */
window.CALABRIO_SCHEDULE = window.CALABRIO_SCHEDULE || {};
window.CALABRIO_SCHEDULE['2025-10'] = {
  timezone: 'America/Toronto', // optional default; schedule times are in this tz
  rules: { allowEarlyReturn: false, requireManagerOverride: true, overridePinHash: null }, // optional policy
  events: [
    // event-per-day (or multiple events per day)
    { date: '2025-10-01', intervals: [
        { start: '11:30', end: '12:00', type: 'lunch', label: 'Lunch', id: '2025-10-01-lunch' },
        { start: '15:00', end: '15:15', type: 'break', label: 'Afternoon Break', id: '2025-10-01-break1' }
      ]
    },
    { date: '2025-10-02', intervals: [ { start: '12:00', end: '12:30', type: 'lunch', label: 'Lunch', id: '2025-10-02-lunch' } ] },
    // ...
  ]
};

Notes:

Dates + times stored in localized strings. The runtime will combine date + time and interpret in specified timezone using the browser Intl API. If timezone absent, assume local system timezone (but warn if mismatch).

rules allow you to configure behavior per-month: whether early returns are permitted, whether overrides require a manager PIN, etc.

Maker (Gemini) will generate these files so you don't need manual entry.

5) Initialization & runtime flow (detailed)

1. Feature load

When user opens the Clock panel (or on app startup if you prefer), call FeatureLoader.load('calabrio') which loads calabrio-core.js and calabrio-ui.js.

calabrio-core reads window.CALABRIO_SCHEDULE for current month key (e.g., 2025-10). If not present, show “No schedule installed” UI.



2. Schedule parsing

Convert every event entry into canonical Date objects with time zone awareness:

Use new Date(Date.UTC(year, monthIndex, day, hour, min)) + offset if timezone != local, or use Intl.DateTimeFormat to get offset.


Store canonical start/end timestamps.



3. Compute next event

From now, compute the next event and the "state" (upcoming, active, past).

Expose calabrio.getNextEvent() and calabrio.isActive().



4. Timers & pre-notifications

Create one short-lived timer for pre-notifications (10 min, 1 min) and one precise timer for start/end using setTimeout with computed deltas (clamped to safe maximum).

Use requestAnimationFrame for small visual updates (e.g., countdown smoothness), but the actual timing is controlled via setTimeout to avoid CPU waste.



5. On start

When an event starts, call calabrio.lockUi(event):

Save current app interaction state to IndexedDB (audit): Diag.pushStep('calabrio:locked', { id, ts }).

Add a top-level DOM overlay element (injected into #app-container) with role="dialog" aria-modal="true".

Add a global event-capture layer that stopImmediatePropagation() for clicks/keyboard events except whitelist keys (Escape for emergency? configurable).

Add a body class calabrio-locked to fade/dim top bar and apply styling.




6. During break

Keep countdown ticking. Provide clear UI controls for allowed actions (I’m Back / Request Override).

Persist state (active event id, startTime) to IndexedDB so reload during break preserves lock.



7. On end

Remove overlay, remove calabrio-locked CSS class, push audit calabrio:unlocked.

Short “Welcome back” animation via Anim module.



8. Override / early return

If policy allows:

I’m Back triggers immediate unlock and logs reason. Notify user and keep audit.


If manager override required:

Show a modal with a PIN input. If PIN match (local hashed), unlock and audit. Or provide a manager approval code flow (manual).




9. Edge: time drift & sleep

On visibilitychange, resume from sleep, or focus events, re-evaluate current time against schedule and enforce lock/unlock as needed.

If system clock changed (detect performance.now() vs Date.now() skew > threshold), warn user and show instructions to sync time.

6) Lock implementation (safe & non-invasive)

This is an in-app UI lock only:

It prevents user interaction with your application by intercepting and blocking events at document level.

It does NOT attempt to modify browser, OS, or security settings (explicitly forbidden).

It is reversible, auditable, and user-controlled by configured rules.



How to implement safely

Add a full-screen <div id="calabrio-lock-overlay" tabindex="-1"> appended to <body> or #app-container.

Add a pointer-events: auto to overlay, and pointer-events: none to the underlying app wrapping container.

Add keyboard global handler that, while overlay active, prevents key actions except allowed ones (Ctrl+Alt+C to open debug console only if allowed, or a configurable emergency key).

Persist overlay state in IndexedDB, so a reload during a locked period re-creates overlay immediately.


Audit logs

Use Diag.pushStep() or DB.set() to store calabrio:locked/unlocked/override events with timestamps and user action reasons.

7) Animations & visual polish (iOS-like)

Visual design principles

Soft blur + semi-transparent overlay (backdrop-filter: blur(8px)), rounded UI elements, large legible numeric timer with subtle shadow.

Animate overlay enter/exit with combined transform + opacity (GPU-friendly).

Micro-detail ideas

Top bar gloom: apply gradient overlay on top bar using CSS variable and animate opacity from 0 → 0.6.

Pulse countdown: last 5 seconds, add a scale pulse to the timer (scale 1 → 1.05 → 1).

Subtle ambient shadow: the overlay background uses a radial gradient to highlight center.


Performance

All animations use transform and opacity only (except backdrop-filter which may be GPU-intensive on some devices — implement a setting to disable blur if on low-spec device).

Use prefers-reduced-motion to reduce or bypass animations.

9) Edge cases & robustness

Machine sleep or hibernation: re-evaluate schedule on visibilitychange or focus.

User changes system clock: detect large deltas Math.abs(Date.now() - (startup + (performance.now() - perfStart))) and warning message; re-sync schedule calculation.

Time zone mismatch: if schedule timezone != system timezone, show a clear badge and convert times to local view; warn user if mismatch.

Missing monthly file: show a safe UI “Schedule not installed for [month].” Provide a “Load month file” instruction (drag & drop or pick local file).

Multiple user profiles on machine: schedule is local to application, stored in IndexedDB. If shared machines require per-user schedules, let your generated file include user identifiers (opt-in).
