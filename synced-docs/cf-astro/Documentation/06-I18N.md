# 06 — Internationalization (i18n)

## Overview

The site supports two languages:

| Locale | Language | Market | URL Prefix | Status |
|---|---|---|---|---|
| `es` | Spanish | Mexico (primary) | `/es/` | Default locale |
| `en` | English | International visitors | `/en/` | Secondary locale |

---

## Routing Architecture

### Astro Built-in i18n

Configured in `astro.config.ts`:

```typescript
i18n: {
  defaultLocale: 'es',
  locales: ['es', 'en'],
  routing: {
    prefixDefaultLocale: true,    // /es/ is visible (not just /)
    redirectToDefaultLocale: false, // No auto-redirect from / to /es/
  },
},
```

### URL Structure

| URL | Page | Locale |
|---|---|---|
| `/` | Redirected to `/es/` via `_redirects` | — |
| `/es/` | Spanish Homepage | es |
| `/en/` | English Homepage | en |
| `/es/booking` | Spanish Booking Page (planned) | es |
| `/en/booking` | English Booking Page (planned) | en |
| `/es/privacy-policy` | Spanish Privacy Policy (planned) | es |
| `/en/privacy-policy` | English Privacy Policy (planned) | en |
| `/api/booking` | Booking API (no locale) | — |
| `/api/ingest/*` | Analytics proxy (no locale) | — |
| `/api/privacy/arco` | ARCO request API (no locale) | — |

### Redirect Configuration

File: `public/_redirects`
```
/ /es/ 301
```

This uses Cloudflare Pages' built-in redirect handling (no Worker compute needed).

## Translation System (`src/i18n/config.ts`)

### Key Functions

#### `getTranslations(locale: Locale): Promise<Record<string, any>>`

Lazy-loads and caches translation JSON files:

```typescript
const translationCache = new Map<string, Record<string, any>>();

export async function getTranslations(locale: Locale) {
  if (translationCache.has(locale)) return translationCache.get(locale)!;
  let messages;
  if (locale === 'es') {
    messages = (await import('./translations/es.json')).default;
  } else {
    messages = (await import('./translations/en.json')).default;
  }
  translationCache.set(locale, messages);
  return messages;
}
```

#### `t(messages: Record<string, any>, key: string): string`

Dot-notation key resolver with missing translation warnings:

```typescript
export function t(messages: Record<string, any>, key: string): string {
  const keys = key.split('.');
  let value: any = messages;
  for (const k of keys) {
    if (value && typeof value === 'object' && k in value) {
      value = value[k];
    } else {
      console.warn(`[i18n] Missing translation: ${key}`);
      return key; // Fallback to key path
    }
  }
  return typeof value === 'string' ? value : key;
}
```

#### `getLocaleFromUrl(url: URL): Locale`

Extracts locale from URL pathname:

```typescript
export function getLocaleFromUrl(url: URL): Locale {
  const [, lang] = url.pathname.split('/');
  if (isValidLocale(lang)) return lang;
  return defaultLocale;
}
```

#### `getLocalizedPath(path: string, locale: Locale): string`

Generates a localized URL path:

```typescript
export function getLocalizedPath(path: string, locale: Locale): string {
  const cleanPath = path.replace(/^\/(es|en)/, '');
  return `/${locale}${cleanPath || '/'}`;
}
```

## How Components Consume Translations

### Pattern: Page → Layout → Component

1. **Page** loads translations:
   ```astro
   const locale: Locale = 'es';
   const messages = await getTranslations(locale);
   ```

2. **Page** passes to layout:
   ```astro
   <MarketingLayout title={t(messages, 'Metadata.title')} locale={locale}>
   ```

3. **Layout** passes to components:
   ```astro
   <Header locale={locale} messages={messages} />
   ```

4. **Component** uses `t()`:
   ```astro
   <h2>{t(messages, 'Services.title')}</h2>
   ```

### Language Switcher

In `Header.astro`, the language toggle computes the alternate path:

```typescript
const altLocale = locale === 'es' ? 'en' : 'es';
const altPath = currentPath.replace(`/${locale}`, `/${altLocale}`);
```

Then renders a link:
```astro
<a href={altPath}>{altLocale.toUpperCase()}</a>
```

## Sitemap i18n

The `@astrojs/sitemap` integration includes locale metadata:

```typescript
sitemap({
  i18n: {
    defaultLocale: 'es',
    locales: { es: 'es-MX', en: 'en-US' },
  },
}),
```

This generates `hreflang` attributes in the sitemap XML.

## SEO Locale Tags

`BaseLayout.astro` generates proper hreflang tags:

```html
<link rel="alternate" hreflang="es-MX" href="https://madagascarhotelags.com/es/" />
<link rel="alternate" hreflang="en" href="https://madagascarhotelags.com/en/" />
<link rel="alternate" hreflang="x-default" href="https://madagascarhotelags.com/es/" />
```

And Open Graph locale:
```html
<meta property="og:locale" content="es_MX" />
```
